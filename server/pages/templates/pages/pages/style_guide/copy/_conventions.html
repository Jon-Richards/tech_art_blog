Conventions

The styles below are based on type selectors (HTML tags) so that the content itself is stylesheet agnostic. 

        
          <div class="copy">
            <h1>Hello World!</h1>
            <p>
              The h1 and this p tag will have copy styles.
            </p>
          </div>

Styles can also be applied in-line using the prefix "copy" followed by a hyphen and the name of the tag, like so: 

        
          <p class="copy-p">
            This paragraph, and only this paragraph, will have copy styling.
          </p>

Intents

In some instances, styling beyond simple type selectors is necessary. For example, a paragraph might serve as an introduction to a larger body of text, or an aside might serve as a warning. It's necessary to communicate these intents to the user visually.

The data-intent convention provides a non-aggressive API for communicating the intent of the markup to the application. HTML data attributes are easily accessed by both CSS and JS, and don't collide with the HTML specification. 

        
          <p class="copy-p" data-intent="intro">
            This paragraph, and only this paragraph, will have copy styling with
            special formatting for intro's.
          </p>


The Intents convention is not a substitute for the ARIA specification or semantic HTML tags and attributes. It's designed as a way to fill in the gaps where a standard or more widely supported API fails to meet the design's needs.
